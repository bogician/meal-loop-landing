## What Went Well

- **CP-1 finally landed — the two-epic landmine is defused.** Reconciling the planning docs Outfit→Manrope was the #1 critical-path action item out of *both* the Epic 1 and Epic 2 retros, carried as a per-story workaround callout through all of Epic 2. It was committed (`7c250c9`) **before** Epic 3's JSON-LD/metadata stories read the brand docs — closing the exact risk both prior retros flagged. Follow-through, at last.
- **The deferred-work ledger works end to end.** Every routed item landed where promised: the unlabeled-nav-landmark (Epic 1 AI-3) → Story 3.1 Task 4; the next-intl `x-default` redirect (deferred-work #37) → 3.2 Task 1; the `SITE_ORIGIN` trailing-slash/empty-env hardening (from the 3.2 review) → 3.3 Task 1; Story 2.3's Twitter-tag scope seam → 3.5. The ledger is a functioning forward queue, not bureaucracy.
- **"Verify against the installed package" graduated from defensive to load-bearing.** In Epic 2 it caught divergences; in Epic 3 some stories were *only* correct because a fact was verified in `node_modules`: 3.5's whole design (omit `images` so the per-locale OG route image merges back) was confirmed against `mergeStaticMetadata` in the installed resolver; 3.3 confirmed `Languages<string>` is a mapped type that rejects `Object.fromEntries` under strict; 3.2 confirmed next-intl's `alternateLinks` default emits a conflicting `Link` header.
- **Single-source + no-deps discipline held under pressure.** Every canonical/hreflang/sitemap/JSON-LD/OG URL derives from one `SITE_ORIGIN`/`localeUrl` source (sitemap hreflang is byte-equal to the metadata alternates); 3.4 and 3.5 added zero catalog keys by reusing `metadata.description`; AR-6 held (schema-dts + OG lib both declined).
- **The "spine pays off" pattern, third confirmation.** 3.2 built the metadata spine and 3.3/3.4/3.5 were near-mechanical reuses of it — the same compounding seen in Epic 1 (i18n scaffold) and Epic 2 (2.1 brand spine).

## What Was Hard

- **The headless verification limit widened to a third class.** Epic 1 left a *responsive* backlog; Epic 2 added *brand-fidelity*; Epic 3 adds *SEO/social external validators* — every one of the five stories ends with an "outstanding human follow-up" (Google Rich Results, schema.org validator, hreflang checker, Facebook/X/LinkedIn/iMessage/Slack/Telegram card validators). It's now a sizeable batched backlog, and the SEO/social half can only run against a live deployment.
- **The hreflang `languages` map duplication grew across files.** One hardcoded `{en, uk, x-default}` literal in `page.tsx` (3.2) became a second byte-identical copy in `sitemap.ts` (3.3). Two literals must now stay in lockstep by hand; a 3rd locale silently desyncs them. The clean fix (`Object.fromEntries`) breaks the `Languages<string>` mapped type under strict, so it was consciously deferred to the locale-addition story — but the debt is real and now spans two files.
- **Doc-vs-reality drift recurred (smaller echo of CP-1).** The architecture doc annotates `[locale]/layout.tsx` as the JSON-LD inject point; 3.4 correctly diverged to `page.tsx` to avoid leaking `SoftwareApplication` onto Epic 5's shared-layout legal pages. The dev caught and documented the variance, but it's the same doc-drift family that produced CP-1.

---

## Key Insights

1. The deferred-work ledger is a real, working mechanism — items routed across 2–3 epics landed precisely. Keep feeding it.
2. "Read the installed package" is now load-bearing, not just defensive: in Next 16, some stories are only correct because behavior was verified in `node_modules`, not assumed.
3. The headless backlog is now three classes (responsive + brand-fidelity + SEO/social validators) converging on one human/browser stage — and the SEO/social half requires a live deploy.
4. Doc-vs-implementation drift is a recurring class (CP-1 font; architecture JSON-LD location). A lightweight "fix the doc when you diverge" habit would stop it compounding.
