## What Went Well

- **The "spine pays off" pattern repeated (Epic 1 insight, re-validated).** Story 2.1 established the house pattern — adapt the locked iOS vector, commit static binaries, wire via Next.js file-convention, generate with one-time offline tooling, add no deps. 2.2 and 2.3 were near-mechanical reuses; 2.3 explicitly took `src/app/icon.svg` as "the exact precedent." Complexity paid once.
- **"Verify against the installed package" caught real Next-16 divergences.** The `AGENTS.md` mandate surfaced two behaviors that contradicted assumptions: 2.2's favicon emits `sizes="48x48"` (not the AC-predicted `"any"`, because the multi-res `.ico` has a determinable largest entry), and 2.3's metadata route does **not** inherit the parent layout's `generateStaticParams` — the OG route shipped as dynamic `ƒ` until a route-local `generateStaticParams()` was added. Both caught and fixed in-session.
- **Adversarial review stayed signal, not noise.** Across three stories: 1 real patch, a handful of genuine decisions (twitter-tag scope seam, scope-guard), and ~35 false positives correctly dismissed via project-aware layers (e.g. byte-identical path data disproving "cutouts fill solid").
- **Clean commit hygiene.** Scope-guard resolutions from each review were actually honored at commit time — verified: `.claude/settings.local.json` churn went into its own `chore` commit (548165d), and no story commit pulled in out-of-scope files.

## What Was Hard

- **CP-1 (docs → Manrope) never landed — the standout continuity miss.** Reconciling DESIGN.md / architecture.md / epics.md / prd.md / project-context.md from Outfit to Manrope was Epic 1's #1 critical-path action item, explicitly "must complete before Epic 2 references the design docs," owned by Bogdan + Paige. It did not happen. Mitigation worked — all three stories carried a prominent "FONT IS MANROPE, NOT OUTFIT" callout and shipped no bug — but the landmine is still armed for Epic 3 (JSON-LD logo + per-locale metadata read the brand/design docs). **Decision this retro: do CP-1 now, before Epic 3.**
- **The literal-hex gotcha needed re-explaining twice.** Favicons (2.2) and OG cards (2.3) render *outside the DOM* and cannot read CSS tokens, so they must bake literal light hex (`#2E7D4F` / `#f5f4f1` / `#1a1918`). Each story needed a callout that this is NOT the forbidden "no hardcoded hex" anti-pattern (which governs components/`globals.css`). Conceptually slippery; worth codifying.
- **Headless env still cannot close visual ACs — and Epic 2 widened the class.** Every story left brand-fidelity checks for a human: mark optical weight at 28px, `.dark` recolor, 16px favicon legibility, OG card render in real validators. This is the same structural limit as Epic 1 (AI-1), but Epic 2 added a *brand-fidelity* class on top of the *responsive* class.
- **Recurring process papercut: `settings.local.json` churn.** Auto-generated Claude Code permission-allowlist churn forced the same "commit it separately" scope-guard decision in all three story reviews, and was dirty in the working tree again at retro time. **Decision this retro: gitignore it (done).**

---

## Key Insights

1. A well-chosen spine story compounds: pay the pattern cost once in story 1, and the rest of the epic becomes near-mechanical reuse. (Re-confirmed from Epic 1.)
2. A skipped doc-reconciliation item does not stay contained — it forces a per-story workaround in every downstream story and keeps rolling forward until someone pays it. CP-1 has now taxed two epics.
3. Some project rules have a sharp boundary that must be stated explicitly or it reads as a violation: "no hardcoded hex" governs the DOM; baked assets (favicons, OG) live outside it and require literal hex.
4. The headless visual-AC limit is structural and now spans two classes (responsive + brand-fidelity). Re-flagging it story by story is waste; it must be batched to a human/browser stage.
