**Constraints.** WCAG 2.1 AA intent (landmarks, keyboard nav, visible focus, ≥44px touch targets, contrast in light AND dark, single `<h1>`, no skipped levels, focus mgmt + escape on mobile menu). CWV: mobile LCP ≤2.5s, CLS ≤0.1, INP ≤200ms; body ≥16px; no horizontal overflow ≥320px. Cookieless analytics (Vercel); no PII/email capture; GDPR+ePrivacy; consent banner must NOT block content, must be revocable, must not feel like a cookie wall. **Stakes.** Consumer-facing acquisition site for a solo-builder iOS product. High polish bar (pixel-aligned to iOS brand, real phone QA, AA, "good" CWV) at modest scale (one page, one CTA, no on-site monetization). Light GDPR/App-Store regulatory pressure.

## 2026-06-19 — Batched decisions (Bogdan)

1. **Mobile menu = full-screen overlay.** Hamburger opens a full-screen calm panel covering the page; large stacked nav links + App Store CTA + locale toggle. Chosen for restfulness, largest tap targets, simplest focus-trap accessibility. Escape + ✕ dismiss; focus returns to the trigger.
2. **Locale switcher = inline `EN · УК` text toggle.** Active locale emphasized (brand color / weight), the other muted. No hidden state, one tap to switch. Appears in nav (desktop) and inside the mobile overlay. Must meet ≥44px target.
3. **Dark mode = OS-driven, no toggle.** Respect `prefers-color-scheme`; dark tokens in `globals.css` activate automatically. No toggle UI in v2. (Upgrades the PRD's "optional stretch" to in-scope because tokens already exist and cost is low.) → Both light AND dark must pass AA contrast + phone QA.
4. **Coming-soon CTA = badge + calm caption.** App Store badge renders as a non-interactive element (not a dead link) with a quiet caption beneath: "Coming soon to the App Store" (localized). When `APP_STORE_LIVE` flips true, the badge becomes the live link and the caption disappears. No email capture / waitlist (out of scope).

**Remaining items → [ASSUMPTION] in draft, surfaced at Finalize:** OG image composition; consent banner exact layout; persisted-preference mechanism (cookie vs localStorage); `prefers-reduced-motion` handling for the loop animation; real-screenshot swap (deferred).

## 2026-06-19 — Accessibility review findings (`review-accessibility.md`)

Reviewer computed WCAG ratios for the locked palette. **Dark mode passes everywhere; light mode has real failures.**

- **B1/B2 (BLOCKER, needs decision):** light brand green `#3D8A5A` fails AA — white-on-green = **4.22:1** (CTA labels), green-as-text on paper = **3.83:1** / on card = **4.22:1** (active locale label, link emphasis). Root cause is the *locked* palette, so it can't be patched only in the marketing layer without either (a) darkening upstream in `globals.css`/iOS source, or (b) changing green's *usage*, or (c) documenting an accepted deviation. Note: fixing green-on-paper to ≥4.5 (the binding constraint, needs light-green luminance ↓ ~20%) also fixes white-on-green. → Escalated to Bogdan as the one phase-blocker.
- **Directly fixed in spines (no decision needed):**
  - B3 — Coming-soon CTA AT semantics specified: static, **non-focusable**, `role="img"` with an accessible name that *includes* "coming soon"; not in tab order; caption reinforces for sighted users. (Not a disabled focusable button.)
  - SF1 — locale toggle: per-label `min 44×44px` + internal padding + a real gap between the two hit areas (verify at 320px).
  - SF2 — focus ring: ≥2px, 2px offset, contrasting halo so it never sits green-on-green.
  - SF3 — device mockups may be `aria-hidden` only because section copy conveys the value independently; post-v2 real screenshots require `alt`/visually-hidden captions.
  - SF4 — active locale + link emphasis get a **non-color** cue (underline + `aria-current`), not green-alone (1.4.1).
  - SF5 — reduced-motion **resolved (was [ASSUMPTION])**: single static frame, no autoplaying cross-fade or loop; any transition must be user-initiated.
  - Locale-switch announcement gap: after a client-side locale swap, place focus sensibly (retain on switcher / move to heading) so the language change is perceivable.
  - Downgraded both spines' unsubstantiated "light + dark verified AA" claim to "dark verified; light pending B1/B2."
- **NICE-TO-HAVE → logged:** N1 light `muted-foreground #6D6C6A` rides the AA line (4.77:1) — consider one step darker (~5:1). N2 Cyrillic gap is aesthetic, not a WCAG failure, but the fallback stack must be Cyrillic-complete. N3/N4 heading structure + overlay focus management confirmed solid.
